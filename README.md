## Branch Strategy
- `main`: stable release branch
- `dev`: integration branch
- `Anh`: personal development branch for Anh
- `Dung`: personal development branch for Dung
- `Long`: personal development branch for Long
- `Khoa`: personal development branch for Khoa

## Workflow Overview
- Each team member develops features on their own personal branch
- Completed work should be submitted to `dev` through Pull Requests
- `dev` is the shared integration branch for the whole team
- `main` is reserved for the most stable and release-ready version
- All commit messages should follow the agreed commit format

## Commit Message Format
Use the following format for commits:

```text
<type>: <short description>
```
Examples:
- feat: add AVL tree insertion
- fix: resolve graph rendering issue
- docs: update team workflow
- refactor: separate visualization logic from data structure logic

## Project Structure
- include/: header files
- src/: source files
- assets/: images, sounds, and other resources
- data/: sample input and data files
- docs/: project documentation
- release/: executable files and final deliverables
